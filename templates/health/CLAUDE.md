# 健康记录员协议

## 项目概述

这是一个**纯记录、不诊断**的个人健康档案子系统，用于记录用户本人及家属的健康状况、就医过程、用药情况与体检数据。

核心定位：
- **记录与整理**，而非提供医疗建议或诊断。
- 所有信息以**结构化、可追溯、可对比**的方式落库。
- `timeline.md` 是全局核心索引，任何新增都要同步。

> 免责声明：本子系统不提供任何医疗诊断或治疗建议，仅做信息记录与整理。任何健康决策请咨询专业医疗人员。

## 目录结构

数据根：`$ME_HOME/health/`

```
$ME_HOME/health/
├── CLAUDE.md              # 本协议
├── README.md             # 子系统说明 + 免责声明
├── profile.md            # 个人健康档案（不常变信息）
├── timeline.md           # 全局倒序时间线索引（核心）
├── conditions/           # 症状 / 病情跟踪
├── visits/               # 就医记录
├── medications/          # 用药记录（当前 + 历史）
├── checkups/             # 体检报告
├── doctors/              # 医生档案与评价
├── insurance/            # 保险与报销信息
├── lifestyle/            # 生活方式（运动 / 饮食 / 睡眠）
└── notes/                # 临时笔记 / 待整理素材
```

各目录职责：
- `conditions/`：按症状或慢性问题分文件跟踪，含表现、规律、就诊计划、治疗记录。
- `visits/`：每次就医一条记录，含日期、医院、科室、医生、主诉、诊断、处方、医嘱、费用。
- `medications/`：当前在用药物（`current.md`）与已停药物（`history.md`）。
- `checkups/`：体检与化验报告，异常高亮、关键数据趋势对比。
- `doctors/`：医生档案，记录就诊体验与评价，用占位姓名。
- `insurance/`：保险类型、报销规则、报销记录。
- `lifestyle/`：运动、饮食、睡眠的长期记录。
- `notes/`：尚未归类的零散信息，整理后再迁入对应模块。

## 文件命名规范

- 就医记录：`visits/<YYYY-MM-DD>-<科室或机构简称>.md`
- 症状跟踪：`conditions/<症状简称>.md`
- 体检报告：`checkups/<YYYY-MM-DD>-<体检项简称>.md`
- 医生档案：`doctors/<机构简称>-<科室>.md`
- 用药记录：`medications/current.md`、`medications/history.md`
- 生活方式：`lifestyle/exercise.md`、`lifestyle/diet.md`、`lifestyle/sleep.md`

命名要求：
- 日期统一 `<YYYY-MM-DD>` 格式。
- 文件名不含真实人名、真实机构全称（用简称或占位）。
- 同一天多条同类记录，追加序号后缀 `-2`、`-3`。

## 信息录入原则

1. **`timeline.md` 是核心索引**：每次新增任何记录（就医、体检、用药变更、症状变化），都必须同步在 `timeline.md` 顶部追加一行（倒序）。
2. **`profile.md` 记不常变信息**：基本信息、过敏史、家族病史、既往病史、手术史、疫苗、紧急联系人。变更时更新对应小节。
3. **就医记录字段完整**：每条 `visits/` 记录应包含——日期、医院、科室、医生、主诉、诊断、处方、医嘱、费用。
4. **体检异常高亮 + 对比**：体检报告中异常项需高亮标注，并与历史数据做趋势对比（如上次值、参考范围、变化方向）。
5. **用药变更**：新增用药更新 `medications/current.md`；停药时从 `current.md` 移除并归档到 `medications/history.md`，记录停药日期与原因。

## Claude 工作方式

- **自动归类落库**：用户口述的健康信息，自动判断归属模块并写入对应文件。
- **多模块同步**：一条信息可能涉及多个模块（如一次就医同时产生 visits 记录、medications 变更、timeline 索引），需全部同步更新。
- **每次写后更新 timeline**：写入任何记录后，立即在 `timeline.md` 追加索引行。
- **模糊信息标 `[待确认]`**：用户表述不清或缺失的字段，统一标 `[待确认]`，不臆测填充。
- **不做医疗诊断，仅记录整理**：不提供诊断结论、治疗方案或用药建议；"分析方向"类内容须明确标注为非医嘱、仅供整理思路。
