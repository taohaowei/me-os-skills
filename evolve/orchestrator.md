# evolve · 主审 / 编排（每周一离线进化）

> 本文件是离线进化任务的总入口，由 `runner.sh` 经 `claude -p` 调用（默认每周一）。
> 目标：把过去一周 `~/.me` 的记忆整理干净（M2）+ 复盘本周对话产出框架改进建议（M3），合成**一份**周报。
> runner 会在末尾注入 `【运行日期】`/`【窗口起始】`/`【数据根】`/`【框架根】` 的**真实值**，并把 M2、M3 的细则**作为附录拼在同一个 prompt 里**——你直接读附录执行，**不需要再去 Read 框架仓里的文件**。

## ⚠️ 占位符规则（最重要）
本文件出现的 `<…>` 都是**待替换占位**。最终写进任何文件（周报、commit message）的内容**绝不允许残留尖括号 `<` `>`**；某项无内容就写"无"。`【运行日期】` 等是已知真实值，直接用，不要照抄中文方括号名本身。

## 总原则（贯穿全程）
- **只整理数据仓 `$ME_HOME`，绝不碰框架仓 `$SKILL_DIR`（skills/me）。** M3 的框架建议只写进周报文字，不改任何 `experts/` / `SKILL.md`。（runner 在你跑完后会**机械校验框架仓零改动**，违反即报警退出。）
- **只归档、不删除**：旧内容压缩后移到 `$ME_HOME/archive/`，绝不 `rm`。
- **对抗核验**：提交前回查磁盘实况——你声称"改了/建了/归档了"的文件必须真实存在且内容一致；落盘前再读一遍确认。
- **SAFETY 不自动降级**：风险等级只能维持或上调；下调一律留给人确认。
- **后台任务**：不打扰用户、不发问、完成即退。

## 步骤

### Step 0 · 前提检查（先看这一周有没有新东西）
- 看 `$ME_HOME/sessions/` 是否有**自【窗口起始】以来**的会话文件；并 `git -C "$ME_HOME" log --since="【窗口起始】"` 看这一周 `~/.me` 是否有改动。
- **若过去一周既无新对话、又无更新** → 写一行「【运行日期】：过去一周无对话或无更新，跳过进化」到 `$ME_HOME/evolution/runs/【运行日期】.md`，**结束，不做后续**。

### Step 1 · M2 记忆整理
按**下方附录【M2 记忆整理细则】**严格执行（处理本周有改动的文件：冷热分层 / 去冗余 / 跨文件一致性 / SAFETY 复评）。

### Step 2 · M3 镜头复盘
按**下方附录【M3 镜头复盘细则】**严格执行（复盘本周对话 → 只产「通用框架改进建议」）。

### Step 3 · 对抗核验 + 本地提交
- 回查 Step 1 改动：archive 只增不删、热文件确实变精简、一致性冲突如实标注、SAFETY 未被擅自降级。
- 确认你**没有改动 `$SKILL_DIR`（框架仓）任何文件**。
- 核验通过 → `git -C "$ME_HOME" add -A && git -C "$ME_HOME" commit -m "evolve: 【运行日期】 周整理"`。**绝不 push、绝不在 `$SKILL_DIR` 里 commit。**

### Step 4 · 写当周唯一周报
写到 `$ME_HOME/evolution/runs/【运行日期】.md`（**一周一个文件**；M3 建议**写在报告里**，不另开文件）。**遵守上面的占位符规则：下面每一项都填真实内容，不留尖括号。** 结构：

- 标题：`进化周报 【运行日期】（窗口：【窗口起始】~【运行日期】）`
- `## 🧹 记忆整理（M2）`：归档/压缩了什么、热记忆约减少多少；发现的跨文件一致性冲突；SAFETY 复评结论
- `## 🔧 框架改进建议（M3，待人工 review）`：每条 = 现象（引对话片段）→ 建议改 `skills/me` 哪个文件哪句 → 为什么是通用而非个人；无则写「本次无框架级建议」
- `## 📌 待人工确认`：需要用户拍板的（如一致性冲突如何对齐）；无则写「无」
