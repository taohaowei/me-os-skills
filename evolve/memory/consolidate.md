# evolve · M2 记忆整理

> 由 `orchestrator.md` 调用。目标：让 `~/.me` 的记忆**干净、专业、不冗长**，重点保护"每次对话都热加载"的那几个文件别变胖。
> 只整理 `$ME_HOME` 下的数据；只归档不删除；写入内容保持 `[事实]` / `[判断]` / `[假设]` 标记。

## 冷 / 热记忆分层（核心）

- **热记忆** = `/me` 启动必读：`MEMORY.md`、`core/PROFILE.md`、`core/STATE.md`、当前活跃的 `threads/`、被提及的 `relations/`。它们一胖，每次对话都变慢变差。
- **冷记忆** = 旧会话细节、被取代的旧信念、已落地的旧议题。

对每个被**近期改动过**的 thread / relation / health 文件：

1. **压缩**：把冗长的历史流水提炼成顶部一段"当前活跃摘要"（保留 L1–L5 结构与关键结论）。
2. **归档**：把被压掉的原始细节**追加**到 `$ME_HOME/archive/<同结构路径>`（如 `archive/threads/xxx.md`），**不被启动加载**。
3. **绝不删除**：archive 只增不减；拿不准就保守保留。

## 跨文件一致性检查

核对硬事实（司龄、年龄、日期、关系、汇报链等）。发现冲突（如 `PROFILE` 写"7 年"、`STATE` 写"8 年"）→ **不擅自改**，在交给 orchestrator 的汇报里列入"待人工确认"，并给出建议口径。

## 索引与快照

- `MEMORY.md`：保持是准确的"地图"，≤ 200 行；删掉指向已归档 / 已失效内容的死链。
- `STATE.md`：覆盖式快照，只反映最新；已沉淀进 thread 的旧洞察从 STATE 移除（它们已在 thread 里）。

## SAFETY 复评（每次都做并留痕）

若近期对话触及自我价值 / 创伤 / 重大丧失等：复评 `core/SAFETY.md`——更新"上次评估日期"并注明"<原因>复评，维持低 / 上调"；需要时补"预警信号 / 需持续关注"。**只能维持或上调，绝不自动下调。**

## 关于 style-notes 与个人档案

`config/style-notes.md`、`PROFILE.md` 等就是**普通记忆文件**，按上面"保持精简、保持准确"维护即可——这不是给 agent"调参"，只是让这些文件保持干净。

## 产出

把"改了哪些 / 归档了什么 / 发现哪些冲突 / SAFETY 结论"汇报给 orchestrator（由它汇进当日日报）。
