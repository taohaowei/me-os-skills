# /me — 本地优先的个人 AI 助手框架

> **一句话定位**：一个本地优先、Markdown 档案驱动的个人 AI 助手框架——亦师亦友，用一个统一人格处理你的**工作 / 心理 / 家庭 / 健康 / 成长**全维度对话。

> ⚠️ **使用前必读 → [DISCLAIMER.md](./DISCLAIMER.md)**
> 本框架**不是医疗器械，不能替代专业心理治疗、精神科诊断或任何临床医疗服务**。健康记录子系统**只记录、不诊断、不开处方**。如遇心理危机或医疗紧急情况，请立即联系专业人员或当地危机热线（见 `config/crisis-hotlines.{{locale}}.md`）。

---

## 它解决什么问题

通用聊天助手不了解"你"——每次对话都从零开始，记不住你的团队成员、家庭关系、健康历史和成长脉络；而它们又倾向于在情绪和健康话题上越界给"诊断/建议"。

`/me` 用三个设计回应这两个痛点：

1. **一个统一人格**陪你走全维度，不需要在"教练 / 咨询师 / 记录员"之间反复切换上下文。
2. **本地 Markdown 档案**沉淀对你的长期理解，越用越懂你——而且数据完全在你自己机器上。
3. **协议前置**：安全协议、三态标记、渐进式深化，约束助手"该记什么、不该断什么"。

## 差异化卖点

| 卖点 | 说明 |
| --- | --- |
| **双仓隔离** | 框架（镜头 / 协议 / 方法论）公开开源；你的真实档案数据私有，存放在 `$ME_HOME`（默认 `~/.me`），**永不进入公开仓**。 |
| **镜头路由 Lens Routing** | 统一人格按话题动态加载不同"专家镜头"（心理咨询 / 人员分析 / 项目诊断 / 资源协调 / 职业规划 / 向上管理 / 健康记录员），**只换镜头、不换人格**。 |
| **L1–L5 渐进式档案** | 对每个人 / 每个议题分 5 级深度，每次只深化 0.5–1 级，避免一次性逼问；所有内容带 `[事实]` / `[判断]` / `[假设]` / `[来源]` 三态标记，区分"知道的"与"猜的"。 |
| **安全协议前置** | 每次启动先查 SAFETY；出现风险信号即触发危机流程，并指向本地化危机热线（`config/crisis-hotlines.{{locale}}.md`）。 |
| **健康记录子系统** | 纯记录、不诊断、不建议的健康档案规范——只留下结构化事实，把判断留给专业人员。 |

```
┌─────────────────────────────────────────────┐      ┌──────────────────────────────────────┐
│  框架仓（本仓 · 公开开源 · Apache-2.0）         │      │  数据根 $ME_HOME（私有 · 默认 ~/.me）    │
│  ── 镜头 / 协议 / 方法论 / 模板 ──             │      │  ── 你的真实档案 · 永不入公开仓 ──        │
│                                               │      │                                        │
│  SKILL.md         统一入口 + 启动/安全协议      │      │  config/user.config   身份与 locale     │
│  experts/         7 个专家镜头                 │ 读取 │  relations/           人员档案 (L1–L5)   │
│  frameworks/      L1–L5 / 三态标记 / 镜头路由   │ ───▶ │  threads/             议题档案 (L1–L5)   │
│  toolkit/         EMERGENCY 等工具协议         │ 写入 │  health/              健康记录            │
│  templates/       各类档案结构模板             │ ◀─── │  sessions/            对话会话记录        │
│  config/          危机热线 / 配置样例          │      │  decisions/           决策记录           │
│  scripts/         init / redact-check         │      │                                        │
│  docs/ examples/  上手指南 / 脱敏示例          │      │  （由 scripts/init 生成，内容仅你可见）    │
└─────────────────────────────────────────────┘      └──────────────────────────────────────┘
        ↑ 可公开 fork / star / PR                            ↑ 留在你本地，不提交、不上传
```

框架是可复用的"引擎"，不含任何个人数据；你的全部真实内容只存在 `$ME_HOME`，与本仓物理隔离。

## 快速上手

1. **获取框架**：fork / clone 本仓到本地。
2. **初始化数据根**：运行 `scripts/init`，它会在 `$ME_HOME`（默认 `~/.me`）创建目录骨架、写入 `config/user.config`，并把 `$ME_HOME` 加入 `.gitignore` 防止误提交。
3. **开始对话**：在支持本技能的客户端里执行 `/me`，或 `/me <你想聊的任何话题>`。不带话题也可以，直接说"先聊聊"。

完整步骤、配置项与常见问题见 **[docs/getting-started.md](./docs/getting-started.md)**。

```bash
# 初始化（在本仓根目录执行）
./scripts/init

# 提交公开仓前，先跑一遍脱敏自检
./scripts/redact-check.sh
```

## 安全与隐私

隐私是本框架的第一性原则，不是附加项：

- **数据永不出仓**：你的真实档案只存在 `$ME_HOME`（默认 `~/.me`），与公开框架仓物理隔离；`scripts/init` 会自动把数据根加入 `.gitignore`。**请勿**将任何真实档案、会话记录或健康数据提交到公开仓。
- **第三方隐私**：档案中会提到家人、同事等第三方。请仅记录与对话目标相关的最小必要信息，避免写入可直接定位到他人的敏感细节；公开任何衍生内容前务必脱敏。
- **不贴临床标签**：框架禁止给任何第三方贴精神病学 / 诊断标签，健康记录子系统只记事实、不下结论。
- **脱敏自检 redact-check**：提交公开仓或分享前，运行 `scripts/redact-check.sh` 扫描真实人名、电话、精确日期、内网地址等疑似敏感信息；占位符统一用 `<...>` 或 `{{...}}`（如 `<姓名>`、`<YYYY-MM-DD>`、`{{user_name}}`）。
- **危机处理本地化**：所有危机热线号码不硬编码在框架里，统一维护在 `config/crisis-hotlines.{{locale}}.md`，按用户 locale 加载。

漏洞与安全问题的披露流程见 **[SECURITY.md](./SECURITY.md)**。
