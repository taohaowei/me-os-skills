# 管理框架速查

> 各镜头常用的核心管理框架速查。详细见 experts/ 下各镜头定义。

---

## 人员管理

### DISC 沟通风格
| 类型 | 特征 | 沟通要点 |
|------|------|---------|
| D（支配型） | 直接、结果导向、果断 | 简明扼要，给选项，不啰嗦 |
| I（影响型） | 热情、乐观、善于表达 | 先认可情绪，给表达空间 |
| S（稳健型） | 耐心、可靠、不喜欢变化 | 给安全感，提前预告变化 |
| C（谨慎型） | 精确、注重细节、逻辑性强 | 给数据和逻辑，允许思考时间 |

### 情境领导力 D1-D4
| 阶段 | 员工状态 | 你的管理方式 |
|------|---------|-------------|
| D1 热情新手 | 有热情没能力 | 指令型：告诉他做什么、怎么做 |
| D2 受挫学徒 | 有点能力但受挫 | 教练型：教+鼓励，高指导高支持 |
| D3 谨慎能手 | 有能力但不自信 | 支持型：给空间，在需要时支持 |
| D4 独立高手 | 有能力有意愿 | 授权型：放手，给方向就够 |

### SBI 反馈模型
- **S**ituation：在什么情况下
- **B**ehavior：你做了什么（具体行为，不评判）
- **I**mpact：产生了什么影响

### GROW 教练对话
- **G**oal：你想达成什么？
- **R**eality：现在的情况是？
- **O**ptions：有哪些选择？
- **W**ill：你打算怎么做？

## 项目管理

### 六维健康度
方向 / 进度 / 流程 / 技术 / 人员 / 干系人

### T-shirt Sizing 估算
| 尺码 | 含义 | 参考 |
|------|------|------|
| S | 半天到一天 | 改个配置、修个小bug |
| M | 2-3天 | 一个完整功能 |
| L | 1周 | 涉及多模块的功能 |
| XL | 2周+ | 需要拆分的大任务 |

### 从混乱到有序四阶段
可见化(1-2周) → 锚定(3-4周) → 规范化(5-8周) → 制度化(9-12周)

### AI 项目双轨制
探索轨（高失败容忍、1-2 周验证单元）/ 交付轨（稳定交付、正常迭代）

### Kill/Pivot 信号
- Kill：2 个周期无法证明可行性 / 技术路线走不通 / ROI 持续恶化
- Pivot：副产品比主产品有价值 / 用户需求根本变化 / 新技术使原方案次优

## 向上管理

### 坏消息汇报 5 步
1. 直接说结论（不铺垫）
2. 事实性背景
3. 影响范围（量化）
4. 解决方案
5. 需要什么支持

### 周报技术→业务转换
| 技术语言 | 业务语言 |
|---------|---------|
| 完成模型fine-tuning | 回答准确率从72%→89% |
| 重构数据pipeline | 处理速度提升3倍 |
| 修复15个bug | 用户核心痛点已全部解决 |

### 争取资源/保卫团队
- 用对方语言框定价值（对 VP 说战略价值，对 +1 说交付加速）
- 量化"不投入的代价"（损失厌恶 > 收益期望）
- 准备 2-3 个选项（不是只有"保"或"砍"）
- 先建 credibility 再要资源
- 小步 pilot，用结果说话

## 个人成长

### 2-2-4-0 时间目标
- 2h 学习
- 2h 管理
- 4h 技术审查
- 0h 编码

### 能力优先级
- P0 生存级：授权、优先级判断、1on1
- P1 效能级：流程设计、跨团队协调
- P2 影响级：技术愿景、人才培养
- P3 领导级：组织设计、文化塑造
