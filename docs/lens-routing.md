# 镜头路由（Lens Routing）

> 本文档说明镜头路由如何工作：从用户话题的关键词路由到「镜头 + 档案」，关系如何从 `$ME_HOME/relations/` 动态发现，以及如何新增一个镜头。
> 内容为纯框架说明，不含任何真实数据。

---

## 核心理念

统一人格（一个「老师」）始终是同一个人，**不切人格**。镜头路由做的只是：根据当前话题，决定**叠加哪个视角**（`experts/` 下的镜头）+ **加载哪些档案**（`$ME_HOME/` 下的真实数据）。

> 镜头 = 视角，不是角色。心理咨询是默认底色，其他镜头叠加其上。镜头融合的方法论见 `methodology.md`。

路由是**隐式的**——用户不需要写任何特殊语法，只要正常说话。系统从话题里识别关键词，自动完成路由。

---

## 路由怎么工作：关键词 → 镜头 + 档案

```
用户输入
   │
   ▼
[启动协议]  ── 加载 user.config、core/ 核心档案、查 SAFETY
   │
   ▼
[意图识别]  ── 提取关键词 / 话题类别 / 情绪信号
   │
   ▼
[查路由表]  ── SKILL.md 路由表：关键词 → 镜头
   │
   ├─▶ 命中一个镜头  → 加载该镜头 + 其默认档案
   ├─▶ 命中多个镜头  → 融合加载（不切人格）
   └─▶ 未命中        → 走默认镜头（心理咨询底色）+ 询问澄清
   │
   ▼
[安全前置]  ── 若含危机信号，优先转入安全协议
   │
   ▼
生成融合回应
```

路由表条目形态（示意，真实条目以 `SKILL.md` 为准）：

| 用户说的话（信号） | 加载的镜头（experts/） | 加载的档案（$ME_HOME/） |
|----------|----------------------|--------------------------|
| `<情绪/压力相关词>` | `<心理镜头>.md` | 当前活跃 thread |
| `<家人相关词>`（按 `relations/<家人分类>/` 实际成员） | `<心理镜头>` + `<家庭系统视角>` | 家庭动力学 thread + `relations/<家人分类>/<对应>.md` |
| `<团队成员>`（按 `relations/<团队分类>/` 实际成员） | `<人员分析镜头>.md` | `relations/<团队分类>/<对应>.md` |
| `<项目/进度/风险相关词>` | `<项目诊断镜头>.md` | 工作 thread + `projects/<项目>/` |
| `<跟上级/业务方沟通>` | `<向上管理镜头>.md` | `relations/<干系人分类>/<对应>.md` |
| `<看医生/吃药/体检/报告>` | `<健康记录员镜头>.md` | 切到 `health/` 工作目录 |

**跨域话题很常见**：一句话可能同时命中多个信号，此时**叠加多个镜头**，由统一人格融合输出——先接情绪，再做分析。

> 危机信号触发时，安全协议优先于路由结果。号码不内联，统一指向 `config/crisis-hotlines.{{locale}}.md`。

## 关系从 `$ME_HOME/relations/` 动态发现

**绝不把人名写死在框架里。** 框架是公开的，里面不能有任何真实人名。关系网完全从数据仓动态发现：

```
$ME_HOME/relations/
├── <家人分类>/    ← 文件名即家庭成员，如 <称呼>.md
├── <团队分类>/    ← 文件名即团队成员
└── <干系人分类>/  ← 文件名即上级 / 业务方等
```

发现与匹配流程：

1. **列目录**：从 `relations/` 各子目录下的**文件名**得知「都有谁」。
2. **按提及匹配**：用户提到某个称呼时，匹配到对应档案文件并加载。
3. **未建档则建档**：提到的人还没有档案，按 L1-L5 渐进式建档（见 `methodology.md`），从 L1 起步。
4. **子目录定镜头**：人在哪个子目录，隐含了默认镜头——团队分类多用人员分析，干系人分类多用向上管理，家人分类多用家庭系统视角。

> 这样设计的好处：换一个用户，关系网完全不同，但框架代码一字不改——人名只存在于私有数据仓 `$ME_HOME`。

## 如何新增一个镜头

镜头是纯框架资产，新增是两步操作（全程不碰个人数据）：

### 第 1 步：在 `experts/` 下加文件

新建 `experts/<新镜头名>.md`，建议包含：

```markdown
# <镜头名>

## 角色定位 / 关注点
<这个视角主要看什么>

## 沟通风格
<语气与现有镜头保持一致，不抢人格>

## 核心方法论 / 框架
<沿用 methodology.md 的三层提问，并补充本镜头特有问题>

## 默认档案
$ME_HOME/<该镜头对应的数据路径>

## 我绝对不做的事
<边界声明>
```

### 第 2 步：在 `SKILL.md` 路由表加一行

在 `SKILL.md` 的「关键词 → 加载镜头」表中追加：

```
| <触发关键词> | experts/<新镜头名>.md | $ME_HOME/<默认档案路径> |
```

补充事项：
- 若新镜头会产生需落档的认知，在 `SKILL.md` 持久化分区表里指明写到 `$ME_HOME/` 的哪个目录、用增量还是覆盖策略。
- 若需要一种新档案结构，在 `templates/` 下加对应空白模板。

> 校验清单：新增完成后确认——（a）框架仓 `git status` 不含任何个人数据；（b）镜头文件内无真实人名（只用 `<...>` / `{{...}}` 占位符）；（c）路由表能从关键词命中新镜头；（d）危机/敏感信息仍只引用 `config/crisis-hotlines.{{locale}}.md`，不写死。

---

## 路由原则小结

- 一切以**意图**为先，关键词只是线索，不是硬规则。
- 多镜头命中时融合，不分裂人格。
- 关系对象动态发现，框架不硬编码任何具体关系。
- 安全协议永远前置于普通路由。
