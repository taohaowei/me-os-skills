# 架构：双仓分离

> 本文档说明本框架的核心设计——「框架公开 / 数据私有」的双仓分离，以及由它支撑的镜头路由与 L1-L5 渐进式档案机制。

---

## 一句话总览

**框架是可复用的引擎，数据是只属于你的内容。** 两者用环境变量 `$ME_HOME` 解耦，分别活在两个永不交叉的目录里：

- **框架仓**：你日常用的 live 技能目录（`skills/me`）。它是**唯一带 git remote** 的仓，可以公开、可以被任何人克隆使用。里面只有镜头、协议、方法论、模板，**不含一个字节的个人数据**。
- **数据仓**：`$ME_HOME`（默认 `~/.me`）。它**没有 remote，永不 push**，是你的真实档案——基本信息、关系网、议题主题线、健康记录、会话历史。它只活在你的本地磁盘上。

这样的分离不是洁癖，而是隐私的结构性保障：框架可以被任意分享、协作、版本化，而个人数据从物理上就没有出口。

## 为什么用 `$ME_HOME` 解耦

框架里**所有指向个人数据的路径，都写成 `$ME_HOME/...`**，从不写死成 `~/.me` 或任何绝对路径。带来三个好处：

1. **可迁移**：换机器、换用户、跑测试时，只要改一个环境变量就能切换整个数据根，框架代码一行不动。
2. **可隔离**：一台机器上可以有多套数据（如真实档案 vs 演示档案），靠 `$ME_HOME` 指向不同目录区分。
3. **防误提交**：框架仓的 `.gitignore` 防御性地忽略 `core/*.md`、`runs/`、`logs/`、`*.local` 等；同时数据从设计上就在框架仓之外的目录，物理上无法被框架仓的 `git add` 捕获。

> 默认值：未设置 `$ME_HOME` 时取 `~/.me`。首次使用时由 `scripts/init` 创建该目录并从 `templates/` 复制空白档案。

## 双仓的生命周期

| 阶段 | 框架仓（live 技能 skills/me） | 数据仓 $ME_HOME |
|------|------------------------------|------------------|
| 初始化 | 提供 `templates/` 与 `scripts/init` | 由 init 创建目录、复制模板、写入 user.config |
| 日常对话 | 被读取：加载镜头 / 协议 / 方法论 | 被读写：加载并增量更新真实档案 |
| 版本管理 | `git commit` / `git push`（公开） | **从不 git push**；如需备份，用本地加密快照，不要加 remote |
| 分享给他人 | 直接克隆即可，干净无隐私 | 永不分享 |

**红线**：任何时候都不要给 `$ME_HOME` 配置 git remote，也不要把数据仓的内容复制进框架仓。框架仓里只允许出现脱敏的 `examples/` 和空白的 `templates/`。

## 框架仓内部分层

- **SKILL.md** —— 统一人格、启动协议、关键词到镜头的路由表。一切从这里开始。
- **experts/** —— 7 个专家镜头。镜头是"视角"，不是"人格"：统一人格按话题叠加镜头，从不切换成另一个角色。详见 `lens-routing.md`。
- **frameworks/** —— 跨镜头复用的方法论沉淀。
- **toolkit/** —— 即用工具，如情绪急救 `EMERGENCY.md`。
- **templates/** —— 各类档案的空白结构模板，复制到 `$ME_HOME` 后填写。
- **config/** —— 危机热线按 locale 分文件（`crisis-hotlines.{{locale}}.md`），以及 `user.config.example`。
- **docs/** —— 本目录的设计文档。

## 与方法论的衔接

双仓分离只解决"数据放哪、谁能看"。"档案怎么长、对话怎么深"由两个机制承载，分别详见：

- **镜头路由（Lens Routing）**：一个统一人格如何按话题加载不同镜头、如何从 `$ME_HOME/relations/` 动态发现关系——见 `lens-routing.md`。
- **L1-L5 渐进式档案 + 三态标记**：每份档案分 5 级深度、每次只深化 0.5-1 级、用 `[事实]/[判断]/[假设]/[来源]` 标注——见 `methodology.md`。
