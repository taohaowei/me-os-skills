# 贡献指南 / Contributing

感谢你愿意为本项目做出贡献。本项目是一个**公开开源的个人助手框架**——我们欢迎对**镜头、方法论、模板**的改进，但对**任何真实个人数据**有零容忍的红线。

请在提交任何 PR 或 issue 之前完整阅读本指南。

---

## 一、可以贡献什么

本仓库只接纳**框架层**的内容，即与任何具体个人无关、可复用的通用资产：

- **专家镜头（experts/）**：新增或改进某个视角的路由规则、提问方式、关注点；
- **方法论（frameworks/）**：L1–L5 渐进式建档、三态标记（[事实]/[判断]/[假设]/[来源]）、安全协议等方法论的完善；
- **模板（templates/）**：人物档案、议题档案、健康记录等**结构模板**（只含字段结构与占位符，绝不含真实内容）；
- **示例（examples/）**：**明显虚构**的演示数据；
- **工具脚本（scripts/）**、**文档（docs/）**、**配置样例（config/）** 的改进。

> 数据根 `$ME_HOME`（默认 `~/.me`）中的用户真实档案**永不进入本仓**。框架文件（experts / frameworks / toolkit / SKILL.md）与本仓同级。

---

## 二、🚫 红线（违反将直接关闭 PR/issue）

本仓库是**公开**的。以下内容**严禁**出现在任何 PR、commit、issue、评论、截图或测试数据中：

- ❌ **任何真实个人数据**：你自己或他人的真实档案、笔记、对话记录、健康记录；
- ❌ **任何第三方姓名**：真实人名、昵称、可被关联到真人的称谓；
- ❌ **公司 / 组织内部信息**：内部项目代号、内网地址、内部系统名、未公开的业务信息；
- ❌ **任何可定位到真人的细节**：真实电话号码、邮箱、住址、精确日期、真实金额、真实病种组合；
- ❌ **第三方精神病学或医学标签**：不得给任何真实或影射的个人贴诊断标签。

所有示例都**必须明显虚构**，并统一使用尖括号或双花括号占位符：

- `<姓名>`、`<YYYY-MM-DD>`、`<议题>`
- `{{user_name}}`、`{{locale}}`

危机热线等本地化信息**不写具体号码**，一律引用 `config/crisis-hotlines.{{locale}}.md`。

---

## 三、提交前自检（必做）

提交任何更改前，**必须**在仓库根目录运行脱敏检查脚本：

```bash
scripts/redact-check.sh
```

- 该脚本会扫描改动，拦截疑似真实个人信息、第三方姓名、内部信息等模式；
- **检查未通过的 PR 不予合并**；
- 即便脚本通过，你仍需自行确认没有任何真实数据——脚本是底线，不是免责。

> 详细的隐私威胁模型与隔离机制见 `SECURITY.md`。

---

## 四、提交流程

1. **Fork & 分支**：从最新主分支拉出特性分支，分支名能说明意图（如 `lens/career-planning`、`template/issue-profile`）。
2. **小步提交**：一个 PR 聚焦一件事；commit message 用中文，清楚说明改了什么、为什么。
3. **自检**：本地运行 `scripts/redact-check.sh`，并自行复查无真实数据。
4. **说明动机**：在 PR 描述中说明这个镜头/方法论/模板解决什么问题、如何复用。
5. **保持中立**：框架内容面向通用场景，不要硬编码任何特定个人、公司或地区的假设。

---

## 五、行为准则

请保持尊重、就事论事。本项目涉及心理与健康等敏感领域，讨论时请注意措辞，避免对任何群体或个人的标签化、污名化表达。

> 提醒：本项目不是医疗或心理服务产品，相关边界见 `DISCLAIMER.md`。
