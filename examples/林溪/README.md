# 示例档案：林溪（全部虚构）

> ⚠️ **本目录全部为虚构演示数据，与任何真实个人无关。**
> 人物、城市、公司、同事、家人、健康与心理细节均为编造，仅用于展示「`/me` 档案填满后长什么样」。
> 请勿将此处任何内容当作真实案例、真实人名或真实模板对待。

---

## 这是什么

这是一套**虚构人物示例数据**，模拟一位用户在 `/me` 框架里使用一段时间后，
`$ME_HOME`（数据根，默认 `~/.me`）目录被填充成的样子。

框架引擎（`experts/` `frameworks/` `toolkit/` `SKILL.md`）与本仓同级、对所有人公开；
而**真实用户的数据存在 `$ME_HOME`，不进开源仓**。本目录是「数据长什么样」的可视化样板。

## 怎么读

1. 先看 `MEMORY.md`——它是所有档案的索引，能让你快速看清全貌。
2. 再看 `core/PROFILE.md`——理解「林溪是谁」，并观察 `[事实]` / `[判断]` / `[假设]` 标记如何使用。
3. 想看一次对话如何沉淀成档案，对照读 `sessions/2025-01-15-mind.md` 与 `threads/认知信念.md`。
4. 想看人物档案的渐进式深化（L1–L5），看 `relations/团队/周楠.md`（标注为 L2）。

> 再次提醒：以上一切均为虚构，仅供理解框架结构使用。
